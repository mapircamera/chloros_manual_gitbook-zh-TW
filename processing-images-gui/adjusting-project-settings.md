# 调整项目设置

在处理图像之前，请务必根据您的工作流程要求配置项目设置。项目设置 <img src="../.gitbook/assets/icon_project-settings.JPG" alt="" data-size="line"> 面板可全面控制校准、处理选项、多光谱指数及导出格式。

## 访问项目设置

1. 在 Chloros 中打开您的项目
2. 点击左侧边栏中的 **项目设置** <img src="../.gitbook/assets/icon_project-settings.JPG" alt="" data-size="line"> 图标
3. “项目设置”面板将显示所有配置选项

{% hint style="info" %}
**设置会随项目自动保存**。当您重新打开项目时，所有设置将自动恢复。
{% endhint %}

***

## 常见工作流的快速设置

### 默认设置（推荐给大多数用户）

对于典型的 MAPIR Survey3 相机工作流程，默认设置效果良好：

* ✅ **暗角校正**：启用
* ✅ **反射率校准**：启用 （需要 MAPIR 校准靶的图像）
* ✅ **去拜耳算法**：标准（快速，中等质量）
* ✅ **导出格式**：TIFF（16 位）

只需导入图像并使用这些默认设置开始处理即可。

***

## 项目设置概述

“项目设置”面板分为多个类别。以下是各部分的概要。完整文档请参阅 [项目设置](../project-settings/project-settings.md)。

### 目标检测

控制 Chloros 如何在图像中识别校准目标。

**关键设置：*** **最小校准采样区域**：目标检测的大小阈值（默认：25 像素）
* **最小目标聚类阈值**：将目标区域分组所需的相似度阈值（默认：60）**何时调整：**

* 若出现误检，请增大采样区域
* 若无法检测到目标，请减小该值
* 若目标被拆分为多个检测结果，请调整聚类阈值

### 处理

主要图像处理和校准选项。

**关键设置：*** **暗角校正**：补偿镜头边缘的暗角 ✅ 推荐
* **反射率校准**：使用校准目标对数值进行归一化 ✅ 推荐
* **去拜耳算法**：将 RAW 转换为 3 通道多光谱的算法
* **最小重新校准间隔**：使用校准目标之间的间隔时间（0 = 使用所有）**高级设置：*** **光传感器时区偏移**：用于 PPK 时间同步（默认：0）
* **应用 PPK 校正**：使用 .daq 文件中的 GPS/曝光引脚数据
* **曝光引脚 1/2**：为双摄像头配置分配相机至曝光引脚

### 去拜耳化方法

目前 Chloros 提供 2 种去拜耳化方法：

#### 标准（快速，中等质量）

标准去拜耳处理速度快，但会出现去拜耳色彩噪点，导致图像精度较低且噪点较多。

#### 纹理感知（慢速，最高质量）\[仅限Chloros+版本\]

纹理感知采用高质量的边缘感知去拜耳算法，结合AI/ML降噪模型，可去除几乎所有的去拜耳噪点。 纹理感知模型运行需要 GPU 内存（VRAM）。建议在可用 VRAM 超过 4GB 时使用，以获得更快的处理速度。

### 指数（多光谱指数）

配置要计算和导出的植被指数。

**如何添加指数：**

1. 点击**“添加指数”** 按钮
2. 从下拉菜单中选择一个指数（NDVI、NDRE、GNDVI 等）
3. 配置可视化设置（LUT 颜色、数值范围）
4. 根据需要添加多个指数

**常用指数：*** **NDVI**：总体植被健康状况（最常用）
* **NDRE**：配合 RedEdge 进行早期应激检测
* **GNDVI**：对叶绿素浓度敏感
* **OSAVI**：适用于可见土壤
* **EVI**：高叶面积指数（LAI）区域**自定义公式（仅限 Chloros+）：**

* 创建自定义多光谱指数公式
* 对所有图像通道进行波段运算
* 保存自定义公式以便重复使用

有关所有可用指数和公式，请参阅 [多光谱指数公式](../project-settings/multispectral-index-formulas.md)。

### 导出

控制输出文件格式和质量。

**可用格式：*** **TIFF (16 位)**：推荐用于 GIS 和科学分析（0-65,535 范围）
* **TIFF (32 位，百分比)**：浮点反射率值（0.0-1.0 范围）
* **PNG (8 位)**：用于可视化的无损压缩（范围 0-255）
* **JPG (8 位)**：文件体积最小，有损压缩（范围 0-255）***

## 保存和加载设置

### 保存项目模板

创建可重复使用的模板以确保工作流程一致：

1. 在“项目设置”面板中配置所有所需设置
2. 滚动至底部的 **“保存项目模板”** 部分
3. 输入描述性模板名称（例如，“Survey3N\_RGN\_Agriculture”）
4. 点击保存图标

**优势：**

* 在多个项目中应用相同的设置
* 与团队成员共享配置
* 确保重复调查的一致性

### 在新项目中加载模板

创建新项目时：

1. 从主菜单中选择 **&quot;新建项目&quot;**

2. 选择**&quot;从模板加载&quot;** 选项
3. 选择您已保存的模板
4. 所有设置将自动应用

### 工作目录

**“保存项目文件夹”**设置指定了新项目默认创建的位置：

* **默认位置**：`C:\Users\[Username]\Chloros Projects`
* **更改位置**：点击编辑图标并选择新文件夹
* **何时更改**：
  * 团队协作用的网络驱动器
  * 存储空间更大的其他驱动器
  * 按年份/客户分类的文件夹结构

## PPK（后处理动态定位）设置

若使用带 GPS 的 MAPIR DAQ 记录仪进行精确定位：

### 先决条件

* 配备 GPS（GNSS）模块的 MAPIR DAQ
* 包含曝光引脚条目的 .daq 日志文件
* 拍摄期间相机已连接至 DAQ 的曝光引脚

### 配置步骤

1. 将 .daq 日志文件放置在项目文件夹中
2. 在“项目设置”中，勾选 **“应用 PPK 校正”** 复选框
3. 如有需要，设置 **“光传感器时区偏移”**（默认：0，即 UTC）
4. 将相机分配至曝光引脚：
   * **单相机**：自动分配至引脚 1
   * **双相机**：手动将每台相机分配至正确的引脚**曝光引脚分配：*** **曝光引脚 1**：从下拉菜单中选择相机型号
* **曝光引脚 2**：选择第二台相机或“不使用”
* 同一台相机不能分配给两个引脚

{% hint style="warning" %}
**重要提示**：必须将曝光引脚正确分配给相应的相机。分配错误将导致地理位置数据不准确。
{% endhint %}

## 高级场景

### 多摄像头项目

在同一项目中处理来自多台 MAPIR 摄像头的图像时：

1. Chloros 会自动检测每台摄像头的型号
2. 每台摄像头将获得相应的处理配置文件
3. PPK：手动将每台相机分配到正确的曝光引脚
4. 所有相机使用相同的导出格式和索引

**示例**：Survey3W RGN + Survey3N OCN 双相机支架

### 延时摄影或多日期测绘

针对同一区域随时间推移的重复测绘：

1. 创建包含标准设置的模板
2. 每次测绘使用一致的校准目标设置
3. 将每个日期作为独立项目进行处理
4. 使用相同的设置以获得可比结果
5. 采用相同格式导出以便进行时间序列分析

### 大型数据集

针对包含大量图像（500+）的项目：

* 考虑按日期或区域拆分为多个子项目
* 使用 Chloros+ 并行处理以加快处理速度
* 考虑使用 CLI 或 API 进行批量自动化处理
* 调整最小重新校准间隔以缩短目标检测时间

开始处理前，请检查以下关键设置：

* [ ] 文件浏览器中已正确检测到相机型号
* [ ] 已启用暗角校正
* [ ] 已启用反射率校准
* [ ] 已导入至少一张校准目标图像
* [ ] 已添加所需的多光谱指数
* [ ] 导出格式符合您的工作流程
* [ ] 已配置 PPK 设置（如果使用包含曝光事件的 .daq 文件）
